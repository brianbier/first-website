Javascript vs. Ruby

Jan 23, 2016 · 4 min read

Ruby's Crazy friend Javascript

Ruby has been my best friend since I started to program and when he introduce me to his crazy friend JavaScript, I said to myself if they are friends then JavaScritp and I will be friends. Trying to learn JavaScript this week has been both fun and painful journey. I've kept constantly comparing JavaScript to Ruby in order to keep my sanity. I came to realize that JavaScript isn't all that different from Ruby, yet the syntax is still unfamiliar and some of the logics are slightly complicated I want to point out a few similarities and differences that caught my attention.

=== vs ==
To start off in JavaScript, there is a 3 equal sign and a double equal sign and they slightly do different things. The 3 equal sign will help you determine if two objects are the same exact object. For example:( 3 === 3 will return true). While the double equal sign in Javascript will determine if the values match. For example:("3" == 3 will return true) because JavaScript will automatically convert the first string into a number in order to make the value match. In Ruby, you accomplish comparison using the double equal sign.

parseInt(); & .to_i
In Javascript in order to convert a string into an integer, you will have to use parseInt("10"); While in ruby you only use the method "10".to_i. The Javascript method is slightly difficult to remember because it does not read well.

Functions vs. Methods
In JavaScript functions are very similar to Ruby methods. The only difference is the syntax. For example in Javascript: 

function string(){
  // print anything to the console using the method below.
  console.log("I am a function in Javascript");
}

// Calling a function to the console
string();


#In Ruby you will create a method like this

def string
  puts "I am a method using Ruby"
end

# Calling a method
string

Each
Iterating through an array or objects is very important in programming. Like Ruby Javascript has its own each function that does the same exact thing. For Example: 

var array = [1,2,3,4,5,6];

array.forEach(function(n){
  console.log(n);
});

#In Ruby you will use the each method
array = [1,2,3,4,5,6]

array.each do |n|
  puts n 
end

Prototypes vs. Classes
Javascript and Ruby are both object-oriented programming languages. Like Ruby classes you can create Javascript prototypes. For Example: 

function myObject(str1,str2,number){
  this.str1 = str1
  this.str2 = str2
  this.number = number
}
// This is how you create an object in JavaScript

var newObject = new myObject("I","am",25)


#In Ruby you create a class like this

class my_object
  def initialize(str1,str2,number)
  @str1 = str1
  @str2 = str2
  @number = number
  end
end

#This is how you create an object in Ruby

new_object = my_object.new("I","am",25)

So if you ever encounter Javascript you will soon realize that it is nothing more than a slightly crazy version of ruby. I hope that these comparisons in syntax help you understand the language better. Good Luck out there and like always keep on Coding.
