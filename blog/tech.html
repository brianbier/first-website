Recursion

Jan 31, 2016 · 4 min read

Another solution for repetition

The idea of recursion is not a commonly used word. So, it seems a bit confusing to the novice programmer trying to understand the idea behind it. I am learning to program and to my understanding Recursion is the idea that a function calls itself. Pretty simple, right? Well not really. Recursion is used to take a big problem and start breaking it down into smaller and smaller pieces ("Divide and Conquer") and continuing to feed their solutions back into the original function until some sort of answer is achieved and the whole chain unwinds. I am a visual learner so I hope that the following example gives you a slight understanding of what recursion does.

For a recursive algorithm to terminate you need a base case (e.g. a condition where the function does not call itself recursively again). 

#Example 1: Counting down

def countdown(n)
# base case which helps break out of the recursion
  return n if n.zero? 
  
  puts n
 # The recursion call testing the following number in the line eg. 5 - 1 = 4
  countdown(n-1)   
end               

countdown(5)
#=>5
#=>4
#=>3
#=>2
#=>1



Example 2: Finding the factorial of a number 

def factorial(n)
# base case which helps break out of the recursion
  if n == 0 
    1
  else
# The recursion call
    n * factorial(n-1)
  end
end

factorial(5) #=>120

Now you may be asking yourself why use recursion when you can get the simple solution using a loop. Well, remember how in ruby there is always an easier way to accomplish somethings. Recursion adds clarity and sometimes reduces the time needed to write and debug code. The problem with recursion is that is is a bit slower and it uses more computer memory. If you are not careful you can get a stack overflow. A Stack overflow is an undesirable condition in which a particular computer program uses more memory space than it has available causing your program to crash.Read this explanation on stack

In conclusion, any problem you can solve recursively you can also solve using the iterators that you know and love. If you find yourself saying "why didn't I just use a while loop here?" then you probably should have. To my understanding there is a balance of when using recursion or not and this is something we are going to learn in Phase 1. Hopefully this explanation gives you a very basic understanding of recursion. Check out the resources I found on recursion that may be helpful for you. Good Luck out there and like always keep on Coding.
