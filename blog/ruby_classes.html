Blueprint of Objects

Jan 17, 2016 · 3 min read

Ruby classes

Classes are very important in the ruby language and using them was very natural for me. A class is a blueprint from which individual objects are created. Creating classes can help you create multiple objects without having to hard code them to each object you create. To help you understand this concept I will create a number guessing game using a class in which you can play with your friend.

First we will create a class. Make sure the class name is capitalized and camel case. 

class GuessingGame

End

#The GuessingGame is considered the blueprint of other objects we will create later.

Next we will create an initialize method. Whenever Ruby creates a new object, it looks for a method named initialize and executes it. So one simple thing we can do is use an initialize method to put default values into all the instance variables. Creating instance variables allow you to have access to that particular variable outside of its own original scope. 

class GuessingGame
  def initialize(number)
     @number = number
  end
end

# The @number is an instance variable and it should be used when you are going to use the variable within multiple methods inside the GuessingGame class.

Now we have to focus on the main body of the program. We need to create a method that will check the number guessed with the original number and give the user hints if they are high or low. 

class GuessingGame
  def initialize(number)
    @number = number 
  end

  def guess(guess_number)
    @guess_number = guess_number
    until guess_number == @number
      if guess_number > @number
        return :high
      else
        return :low
      end
    end
  return :correct
  end
end

#We created an instance variable for the guess argument which is the number the user will enter to guess the correct number. See how we used the @number instance variable to go out of its scope and inside the guess method, that is a great example using the instance variables. You have to include the return keyword for both the low and high symbols because ruby will implicitly return the correct symbol since its the last line in the method.

Finally, we will create a method that compares the two numbers to see if you guessed the correct number 

class GuessingGame
    def initialize(number)
      @number = number 
    end

    def guess(guess_number)
      @guess_number = guess_number
      until guess_number == @number
        if guess_number > @number
          return :high
        else
          return :low
        end
      end
      return :correct
    end

  def solved?
    @guess_number.eql? @number 
  end
end

#The solved? method ends in a question mark, which gives you an idea that the return value will be a Boolean. I use the method .eql? which is just comparing the two instance variables to one another.  If you want to test this code you can use the driver code below. 

#To create an object of the class you use the following code
#game = GuessingGame.new(6) 

#As soon as the object is created you can call the methods within that class like the examples below. 
The (#=> is the output that will occure from calling on the method.)

p guess_returns = game.guess(3) #=> :low
p game.solved? #=> false
p guess_returns = game.guess(7) #=> :high
p game.solved? #=> false
p guess_returns = game.guess(6) #=> :correct
p game.solved? #=> true

In summary, Instance variables are available only for instances of a class. They look like @name. Instance variables are available outside of their scope but within the class blueprint. The initialize method is what actually constructs the data structure. Every class must contain an initialize method. I hope you enjoy creating your first guessing programming game. Now to challenge yourself. I would recommend trying to create a loop outside the class, which lets you continue playing the game until the user inputs the correct number.
